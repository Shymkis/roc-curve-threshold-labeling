            Hi Joseph, nice post.

            I have used pROC out of convenience, but at some point I had to repeat pROC based calculations for many models and large data sets and decided to write my own to speed things up. Being slow is one of the known downsides of pROC. I can't really talk to other packages, but here is a base R implementation using order and cumsum:

            simple_roc <- function(labels, scores){
            Labels <- labels[order(scores, decreasing=TRUE)]
            data.frame(
            FPR=cumsum(!Labels)/sum(!Labels),
            TPR=cumsum(Labels)/sum(Labels))
            }

            It would be interesting to see a timings comparison the get the barest ROC curve using these packages.
